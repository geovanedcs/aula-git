Bem-vindo ao laboratório de prática

Este portal foi desenhado para funcionar como um material de aula completo: possui texto suficiente para gerar alterações reais, componentes reutilizáveis e pontos claros de edição para exercícios de Git.

O conteúdo mistura instruções, listas, tabelas e blocos explicativos, de modo que os alunos possam testar commits pequenos, refatorações, conflitos simples e navegação pelo histórico sem depender de um projeto complexo demais.

Dica de aula

Troque o tema, o texto e as cores em etapas pequenas. Assim, o histórico fica legível e os estudantes percebem exatamente o impacto de cada mudança.

1. Contexto do projeto

O objetivo é ter uma base realista, mas simples de entender. O tema “desenvolvedor iniciante” permite substituir o texto com facilidade para outros cenários, como um portal de notícias, uma enciclopédia curiosa ou uma revista de tecnologia.

2. Lista de tarefas para os alunos

Checklist ordenado

  1. Clonar o repositório e abrir no editor.
  2. Explorar a estrutura de pastas e arquivos.
  3. Fazer uma alteração pequena e commitá-la.
  4. Editar um bloco de conteúdo e observar o diff.
  5. Resolver um conflito simples em dupla ou trio.

Boas práticas

  • Commits pequenos e descritivos.
  • Mensagens claras com verbo no imperativo.
  • Separar HTML, CSS e JavaScript.
  • Testar a interface antes de enviar.
  • Documentar mudanças importantes no próprio código.

4. Agenda da prática

Sequência recomendada

Primeiro, o professor apresenta o layout. Depois, cada estudante faz uma alteração em um trecho diferente da página. Por fim, o grupo compara os resultados, registra commits e revisa os impactos no histórico.

Possíveis exercícios

Substituir os textos de cabeçalho, mudar as cores da interface, adicionar uma nova seção de conteúdo e alterar a tabela de dados são tarefas ideais para praticar fluxo de trabalho com Git.
